Final Project: Pokemon Demo

Sara Maemoto



Overview

For this final project I worked on a team of 3. We created a demo Pokemon game with a few variations. Due to time constraints we were not able to implement all aspects that we wanted to. I created 3 scenes in the project, the beginning scene, the battling scene, and the hospital scene. The hospital scene was a recent fix, I implimented it in order to allow the game to go full circle.

Start Scene

The start scene allows the player to choose their initial Pokemon. It also creates objects that are passed between scenes. Those 4 objects are a backpack which hold items, a team which hold your Pokemon, a scene manager which helps with switching between scenes and remembering previous scenes, and Pokemon which hold the information for all the different types of Pokemon. A short demo reel showing functionality is provided below.

I created 4 different game objects before I knew about the unity function "GameObject.FindObjectOfType". Within the code for the information for all the Pokemon, I created a struct to hold the necessary info for each Pokemon. I initally created an array of stats that made up the Pokemon team, but later changed this to a list. This helped to make other parts of the code dynamic. An example of this change can be scene in the battle scene.

Battle Scene

This is the part of the game where the player uses their Pokemon to fight wild Pokemon. The player is able to catch, run, or fight the wild Pokemon. I created all the animations and functions within the scene. I take no credit for any of the art, all art came from a group member or from the internet.

Hospital Scene

Originally this scene was an unfinished part of the project. Playing through the game I found that when you lost a battle nothing really happened. It gave me a sense of incompleteness, so I decided to finish the scene. All art is credited to a group member and the internet. I created the animations and dialog. The harshness of this scene was incorporated because of extra game ideas we were not able to implement. These ideas are further explained in the extra section of this report.

Extra

While creating this game we had many ideas as to how to make the game more fun. One of our stronger game ideas was to treat Pokemon as mercenaries. The idea was that the player would persuade the pokemon to join them through: money, berries or their renown. The contract would have an expiration date, but the player would also get a chance to renew the contract. This idea lead to a relationship point system, where the Pokemon may start to like the trainer and will no longer need a contract at all. This feature got a lot of discussion because we believed it solved some core mechanic issues for the Pokemon series. It gives Pokemon a sense of realism, rather than being thought of as simply property. It also would create a solid foundation for a rich economic system. The hospital scene slightly shows the tone we were hoping for with our ideas. A dark, but fun game with many different aspects to explore.
